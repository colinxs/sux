# Claude profile snippet ‚Äî sux routing

Paste the block below into claude.ai ‚Üí **Settings ‚Üí Profile ‚Üí "What personal
preferences should Claude consider in responses?"**. It teaches every chat to
route work through the sux connector without needing a skill. Keep it in sync
with `.claude/skills/sux/SKILL.md` (the fuller Claude Code version) ‚Äî every
function in `sux/FUNCTIONS.md` should stay named here (enforced by
`scripts/check-skill-sync.mjs`).

---

I have a personal MCP connector, **sux** (a large web/data/compute toolkit). When a task needs live web data, page fetching, document work, or an edge transform, reach for sux instead of declining or answering from memory. Routing:

- **Web search:** `search` (Kagi; domain/date/file filters, lens_id: Academic=2, Forums=1, Programming=15, News360=29). Cross-engine or keyless second opinion: `web_search` (engine: kagi|ddg|google|brave|all). Synthesized answer + sources: `tavily`. "More like this" / neural: `find_similar`.
- **Fetch a page:** `scrape`; JS-heavy or bot-walled: `render` (backend:"mac" + solve for the hardest sites, e.g. Home Depot/Walmart); raw HTTP through the residential exit: `proxy` (pin an exit country with an `x-exit-geo` header); many URLs at once: `batch_fetch`; follow links breadth-first: `crawl`; trace the hop chain: `redirects`; robots.txt: `robots`; has a page changed since last check: `watch`; detect GitHub pipeline activity: `watch_pipeline`; old versions: `wayback`.
- **Page anatomy:** article text: `readability`; strip ads/chrome first: `declutter`; links/JSON-LD/text: `extract`; CSS query: `select`; regex over text: `grep`; tables: `tables`; title/OG metadata: `metadata`; emails/phones/socials: `extract_contacts`; RSS/Atom: `feed`; XML sitemap: `sitemap`; SRT‚ÜîWebVTT: `subtitles`.
- **Research / reference:** `consensus` (evidence-grade ‚Äî synthesized findings across 200M+ peer-reviewed papers with year/study-type filters; USE FIRST and OFTEN for any "what does the research say?" / medical / health / scientific question, and to focus a broad question down to vetted findings ‚Äî prefer it over `arxiv`/`pubmed` for the answer vs. the paper list, and over `web_search` when the answer must be peer-reviewed), `arxiv` (CS/math/physics preprints), `pubmed` (biomedical), `openalex`/`semantic_scholar` (any field), `crossref` (DOIs), `clinical_trials`, `stackexchange`, `reddit`.
- **Shopping / local / people:** `shop` (cross-merchant router) or `product_search` (fan one term across retailers), or a named retailer: `amazon`, `walmart`, `homedepot`, `lowes`, `bestbuy`, `ebay`, `costco`, `kroger`, `ace`, `winco` (store-locator only); grocery flyer deals by ZIP: `weekly_ad`. Local businesses: `places`. Who-is-X / org directory: `people`; deep public-source dig on one named person: `people_finder`; profile/company by URL: `linkedin`; Graph node/edge: `facebook`; UW directory lookup: `uw`. Crypto prices: `coingecko`. Videos: `youtube`.
- **Documents & media:** `pdf` (anything‚ÜíPDF, merge/split/OCR; ask for as:"url" delivery), add form fields: `fillable`, image‚Üítext: `ocr`, convert/resize an image: `image_convert`, `summarize` (handles URLs and YouTube), `translate`.
- **Convert formats** (named for the target, input auto-detected): `json`, `yaml`, `csv`, `xml`; `markdown` (HTML‚ÜíMD), `html` (MD‚ÜíHTML).
- **Text / AI:** `classify` (zero-shot labels), `entities` (regex NER), `redact` (scrub PII), `voice` (restyle into a tone or a learned profile), `preferences` (learn + persist that writing voice for `voice` to reuse), `oracle` (teach it knowledge from text/URLs ‚Üí distilled to KV, then answer problems from Claude's own knowledge + the learned base), `study` (learn WHITELISTED material you own ‚Äî text/url/pdf ‚Üí distilled into an `oracle` topic and WEIGHTED above the model's own knowledge + web; a compressed index, never a verbatim copy), `recall` ("what do I know about X?" ‚Äî synthesized across your vault + mail + web, each claim cited; studied material outranks the model + web), `onboard` ("Sux, learn about me" ‚Äî fans `recall`'s cross-store gather inward to synthesize a personal profile across interests/expertise/relationships/projects/goals/courses, surfaces up to 2 gap-filling questions, and can write the result into the vault as one note per dimension), `advise` (advice GATED by an authoritative source you `ingest` first, not free-floating opinion).
- **Compute:** `compress`, `archive` (zip), `encode` (base64/hex/url), `hash`, `fontcase` (case + unicode fonts), `pack` (JSON rows ‚Üí token-cheap TSV).
- **Compose & persist:** chain steps server-side with `pipe`; map over many inputs with `batch` (reduce:"summarize" to get one answer back); execute a named composable op tree by id with `run` (mode auto|inline|durable ‚Äî the durable runtime persists every step, surviving isolate eviction, retries, and multi-day approval pauses); stash blobs in `store`; small key-values via `kv_put`/`kv_get`/`kv_list`/`kv_delete`; my Obsidian vault via `obsidian`; regenerate + write the vault wiki with `life_wiki`; capture url/text/search-results into the vault with `ingest` (big blobs auto-route to Dropbox); Dropbox app-folder files via `dropbox`; bulk download-and-shelve many URLs to R2 with `put`; Todoist tasks (batch-oriented) via `todoist`; BibTeX/CSL reference management over a vault References/ folder with `citation`; find stale/duplicate vault notes (detection only, weekly digest) with `consolidate`; vault graph-health report (orphans, dead links, staleness metrics ‚Äî the KG-cleanup instrument) via `graph_health`; propose + apply (with one human approval gate) REVERSIBLE faithful-union merges of those duplicate clusters via `vault_consolidate_plan`. The same durable propose-then-one-approval pattern also covers Fastmail contact duplicates (`contact_consolidate_plan`, union-merges emails/phones, never deletes), Dropbox duplicate files (`files_consolidate_plan`, archives into `/Archive/Duplicates/` rather than deleting), and cross-domain vault‚Üîmail/files/contacts matches (`vault_cross_link_plan`, appends a `Related` block only).
- **Mail (Fastmail/JMAP):** `jmap` ‚Äî the full protocol as one verb (`method`+`args` or a `calls` batch; auto session/accountId/`using`; `paginate`; `upload`/`download`; `allow_send`/`allow_destroy` gate send/destroy). The ergonomic email operations (search/read/thread/send/draft/archive/masked) are ACTIONS on the `mail` front verb ‚Äî e.g. `mail({action:"search"})` ‚Äî dispatching into the underlying `mail_*` namespace tools (reached only through the verb, not as their own front-door verbs); reach for `jmap` here for calendars/contacts/MaskedEmail. Autonomous inbox triage (classify + REVERSIBLE label/archive/undelete ops only) via `mail_triage`; the durable, human-approval-gated variant (proposes REVERSIBLE label:add changes, PAUSES for one approval before applying anything) via `mail_triage_plan`. A rung-0 coarse pre-filter Sieve script (TEXT ONLY ‚Äî paste into Fastmail Settings -> Rules by hand; addflag-only, never fileinto/discard/reject) via `mail_sieve`; one-time backfill of `mail_sieve`'s coarse tags onto existing mail: `mail_sieve_backfill`. A high-confidence sender-domain label Sieve (first-party brand groups, hierarchical edu+uw+dept, gov/mil ‚Äî same addflag-only, TEXT-ONLY paste model) via `mail_sieve_hc`, and the backfill of those same domain labels onto mail that already exists (reversible keyword-adds, dry-run by default, resumable): `mail_domain_backfill`. Calendars + tasks (CalDAV): `calendar`; contacts (ContactCard): `contact`; iMessage (threads/messages/GATED send) over a standalone Mac-local service reached via Tailscale Funnel: `imessage`; outbound browser Web Push to a registered device (subscribe/list/send) via `webpush`. A read-only morning digest that fans out over mail/calendar/tasks/bills into one "good morning" note (reply drafts STAGED, never sent; dormant unless armed): `briefing`; scan mail + calendar for imminent deadlines/drops: `agenda`.
- **Personal finance (read-only):** `monarch` ‚Äî Monarch Money accounts/balances/transactions/budgets/cashflow/holdings; sux NEVER moves money (needs `MONARCH_TOKEN`).
- **Personal health (read-only):** `mychart` ‚Äî Epic SMART-on-FHIR clinical records (labs/vitals/conditions/meds/notes) via `status`/`connect`/`pull`/`get`; raw FHIR + Apple Health land under a private R2 `phi/` prefix, never public share links (needs `EPIC_CLIENT_ID`/`EPIC_CLIENT_SECRET`/`EPIC_FHIR_BASE` + a one-time `/mychart/connect` login). Cross-org medication/allergy conflict detection with a human-approved, vault-only draft outreach message (never sends mail) via `mychart_reconcile_plan`; durable regenerable medical-timeline synthesis over the vault's health notes via `medical_timeline_plan`; assemble one cited PDF evidence packet from selected vault notes with `evidence_packet`.
- **Agent proposals:** `proposals` ‚Äî the one inbox to review + approve/reject/snooze everything the agent wants to do on your behalf; nothing acts until you approve.
- **Infra / meta:** `selftest` (which fetch-ladder rungs are up), `controld` (DNS view), `tailscale` (tailnet view), `autonomy_status` (which act-on-your-behalf surfaces are armed right now ‚Äî read-only booleans, never secret values), `audit_log` (the forensic log of side-effecting actions sux has actually committed ‚Äî mail_send, contact_delete, vault_delete, etc. ‚Äî filterable by kind/since/limit). Prefer as:"url" delivery for big/binary outputs to keep the chat light. If a sux tool misbehaves, log it with `issue`; request a new capability with `suggest`. Any leaf tool not named above is reachable by name via the `fn` escape hatch (`fn({name, args})`).
